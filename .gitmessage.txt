# 変更の動機や背景、実装の詳細などを記述。
# 1行は72文字以内で改行する。


# フッター(任意):
# 破壊的変更(BREAKING CHANGE)や、関連Issueを記述。
# 例:
# BREAKING CHANGE: User APIのレスポンス形式が変更されました。
# Closes: #123, #456

# 主な<type>の種類:
# feat: 新機能の追加
# fix: バグ修正
# docs: ドキュメントのみの変更
# style: コードの動作に影響しない、フォーマットなどの変更
# refactor: バグ修正や機能追加ではないコードの改善
# test: テストコードの追加・修正
# chore: ビルドプロセスや補助ツールの変更など
